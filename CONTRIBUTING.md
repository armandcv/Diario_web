# Guía de Contribución

¡Gracias por tu interés en contribuir a Diario Web! 🎉

## Cómo Contribuir

### 1. Reportar Bugs
- Abre un [issue](https://github.com/tu-usuario/diario-web/issues/new)
- Describe el problema detalladamente
- Incluye pasos para reproducirlo
- Comparte tu entorno (navegador, SO, versión de Node)

### 2. Sugerir Mejoras
- Abre un [issue](https://github.com/tu-usuario/diario-web/issues) con la etiqueta `enhancement`
- Explica por qué sería útil
- Proporciona ejemplos de uso

### 3. Enviar Pull Requests

#### Preparación
```bash
# 1. Fork el repositorio
# 2. Clona tu fork
git clone https://github.com/tu-usuario/diario-web.git
cd diario-web

# 3. Crea una rama
git checkout -b feature/tu-feature
```

#### Desarrollo
```bash
# 4. Instala dependencias
npm install

# 5. Haz tus cambios
# 6. Prueba tu código
npm start

# 7. Commit
git commit -m "Add: descripción clara de cambios"
```

#### Envío
```bash
# 8. Push a tu fork
git push origin feature/tu-feature

# 9. Abre un Pull Request en GitHub
```

## Estándares de Código

- **JavaScript**: Vanilla JS, sin frameworks complejos
- **HTML**: Estructura semántica
- **CSS**: Nombres de clases claros, reutilizable
- **Comentarios**: Solo donde sea necesario
- **Convención**: camelCase para variables/funciones

## Proceso de Review

1. Un mantenedor revisará tu PR
2. Se pueden solicitar cambios
3. Una vez aprobado, se fusionará
4. ¡Tu contribución estará en la siguiente versión!
